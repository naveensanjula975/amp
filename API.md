## Endpoints

### `POST /webhook`
Receives TradingView alert payloads and dispatches trade orders.

**Authentication:** Secret token inside the JSON body (`secret` field).

#### Request Body
```json
{
  "secret": "your_super_secret_webhook_token_here",
  "action": "buy",
  "symbol": "ES",
  "quantity": 1,
  "order_type": "market",
  "price": null,
  "stop_loss": 5100.00,
  "take_profit": 5250.00,
  "strategy": "Futures Automated Strategy MVP",
  "timestamp": "2026-03-31T21:00:00Z"
}
```

## Supported Action Values

| Action | Behavior |
|---|---|
| `buy` | Opens a long position via the broker API |
| `sell` | Opens a short position via the broker API |
| `close` | Closes the open position for the specified symbol |
| `close_all` | Closes all open positions for the symbol (MVP: same as `close`) |

## Supported Order Types

| Order Type | Required Extra Field | Description |
|---|---|---|
| `market` | â€” | Fills immediately at current market price |
| `limit` | `price` | Fills only at specified limit price or better |
| `stop` | `stop_loss` | Triggers a market order when price hits the stop level |

## TradingView Alert Setup

In TradingView, when creating an alert:

1. Set **Webhook URL** to: `https://your-domain.com/webhook`
2. Set **Message** to the JSON generated by the Pine Script `generateAlertBody()` function:
   ```
   {{strategy.order.alert_message}}
   ```
   This automatically uses the structured JSON built into the strategy's `alert_message` parameter â€” no extra configuration needed.
